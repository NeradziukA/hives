# Рефайнмент — Туман войны

## Контекст

Геолокационная игра Hives. Игроки не должны видеть всю карту и всех участников — только то, что находится в зоне видимости их персонажа и союзных локаций.

---

## Концепция

Видимость формируется из трёх источников:

1. **Личная видимость** — радиус вокруг самого игрока, зависит от атрибута Зрение.
2. **Видимость локации** — каждая союзная локация (убежище, база, улей) раскрывает сферу вокруг себя.
3. **Общая разведка фракции** — агрегированная видимость всех союзников в группе (опционально, для поздних фаз).

Всё, что не попадает ни в один из этих радиусов — скрыто туманом. Туман не блокирует реальную карту, но скрывает юнитов, локации противника и динамические события.

---

## Видимость

Атрибуты и навыки, влияющие на видимость (Зрение, Маскировка, Зловоние, Интеллект, радиусы локаций) — см. [calculations/vision.md](../../docs/calculations/vision.md).

### Вражеские локации

- По умолчанию **не видны** в тумане войны.
- Становятся видимы, если союзный игрок находится в зоне видимости локации **И** его радиус перекрывает территорию локации.
- После того как союзник покинул зону — локация отмечается «последним известным положением» и остаётся на карте до следующего визита

---

## Последнее известное положение (Last Known Position, LKP)

- Когда противник или вражеская локация выходит из зоны видимости — на карте остаётся полупрозрачная метка с таймером.
- Таймер затухания: **10 мин** для юнитов. Локации LKP **не затухают** — остаются на карте до следующего визита.
- LKP не обновляется в реальном времени — это «снимок» момента обнаружения.
- Высокий Интеллект уменьшает погрешность снимка (меньше `intel_delay` → более актуальные координаты).

## Знакомая территория (Familiar Territory)

### Концепция

Юнит накапливает «знание местности» в зонах, где часто бывает или проводит много времени. Чем лучше знакомство с местом — тем дольше туман не возвращается после ухода.

Это создаёт асимметрию: своя территория остаётся открытой надолго, чужая — быстро затягивается туманом.

### Источники знания

| Источник                        | Прирост знания              |
| ------------------------------- | --------------------------- |
| Нахождение в зоне               | +1 за каждую минуту         |
| Посещение (вход в зону)         | +5 единоразово              |
| Союзная локация в зоне          | +10 единоразово при захвате |

### Шкала знания и время затухания LKP

Знание хранится на сервере в виде числового значения на ячейку геосетки (hex или grid).

| Уровень знания | Накоплено единиц | Время затухания тумана |
| -------------- | ---------------- | ---------------------- |
| 0 — незнакомо  | 0                | 0 (туман мгновенный)   |
| 1 — замечено   | 1–19             | 10 мин                 |
| 2 — знакомо    | 20–99            | 2 ч                    |
| 3 — освоено    | 100–499          | 12 ч                   |
| 4 — родное     | 500+             | 72 ч                   |

### Маршруты

Маршрут — линейная последовательность ячеек, по которым юнит регулярно перемещается. Сервер автоматически определяет маршрут, если игрок проходит через одни и те же ячейки **не менее 3 раз**.

- Ячейки маршрута получают бонус **+5 единиц знания** за каждое прохождение.
- На карте маршрут отображается как полупрозрачная нить сквозь туман.
- Туман вдоль маршрута затухает медленнее (время = уровень знания ячейки × 1.5).

### Деградация знания

Знание постепенно уменьшается, если юнит не посещает зону.

- Деградация: **−1 единица за 24 ч** бездействия в зоне.
- Ниже нуля не опускается.
- Деградация не применяется к ячейкам с союзной локацией.

### Серверное хранение

```
// Per-player knowledge map entry
{
  cell_id: string,       // geogrid cell identifier
  knowledge: number,     // accumulated knowledge units
  last_visit: timestamp, // for degradation calculation
  is_route: boolean      // marked as frequent route
}
```

## Серверная логика

### Фильтрация событий

Сервер **не отправляет клиенту** данные о юнитах и локациях, которые не попадают в его текущую видимость. Это исключает читерство через перехват трафика.

```
// Pseudocode: server-side visibility filter
function getVisibleEntities(player, allEntities):
  visibleZones = computeVisibilityZones(player)
  return allEntities.filter(e => isInsideAnyZone(e.position, visibleZones))
```

### Вычисление зон видимости

Зоны видимости пересчитываются:

- При движении игрока (новый WS-пакет с координатами)
- При захвате/потере союзной локации
- При применении навыков Маскировка / Зловоние

### WS-сообщения

| Событие               | Направление     | Описание                                       |
| --------------------- | --------------- | ---------------------------------------------- |
| `visibility_update`   | server → client | Полный список видимых зон (полигоны или круги) |
| `entity_enter_vision` | server → client | Юнит / локация вошли в зону видимости          |
| `entity_exit_vision`  | server → client | Юнит вышел из зоны; передаётся LKP-метка с таймером 10 мин |
| `lkp_expire`          | server → client | LKP-метка юнита устарела, удалить с карты (локации не затухают) |

## Клиентский рендеринг

### Слой тумана

- Поверх карты отрисовывается полупрозрачный тёмный оверлей (canvas layer).
- Зоны видимости «вырезаются» из оверлея (composite operation `destination-out` или аналог).
- Границы зон — мягкие (градиент), чтобы избежать резких краёв.

### LOD (Level of Detail)

- При маленьком зуме (много территории на экране) зоны видимости отображаются как кружки/пятна.
- При крупном зуме — детализированные контуры со сглаживанием.

### Производительность

- Пересчёт оверлея только при изменении зон видимости, не каждый кадр.
- Зоны кэшируются как Path2D-объекты.

## Взаимодействие с другими механиками

| Механика                 | Влияние на туман войны                                                       |
| ------------------------ | ---------------------------------------------------------------------------- |
| Бой                      | Атаковать можно только видимого противника                                   |
| Квест «посетить локацию» | Локация должна быть обнаружена (хотя бы LKP)                                 |
| Боты                     | Подчиняются тем же правилам видимости                                        |
| Скопление игроков        | Локация становится видима сумме способностей всех союзников и союзных зданий |
| Лидерство                | Сумма Лидерства в группе даёт бонус к общему радиусу видимости               |

## Фазы реализации

### MVP (Фаза 1)

- [ ] Личный радиус видимости на основе атрибута Зрение
- [ ] Серверная фильтрация: не отправлять скрытые сущности клиенту
- [ ] Клиентский canvas-оверлей с вырезкой зоны игрока
- [ ] LKP-метки для юнитов (таймер 10 мин)

### Фаза 2

- [ ] Видимость союзных локаций (радиусы по таблице)
- [ ] LKP-метки для локаций (постоянные, без таймера)
- [ ] WS-события `entity_enter_vision` / `entity_exit_vision`
- [ ] Навык Маскировка

### Фаза 3

- [ ] Навык Зловоние
- [ ] Интеллект → задержка разведданных
- [ ] Видимости внутри группы с учетом Лидерства
- [ ] LOD-рендеринг зон при масштабировании карты

### Фаза 4 — Знакомая территория

- [ ] Геосетка: разбивка карты на ячейки, хранение knowledge per player
- [ ] Накопление знания: +1/мин нахождения, +5 за визит, +10 за союзную локацию
- [ ] Таблица уровней знания → время затухания тумана
- [ ] Деградация: −1/24ч без посещения
- [ ] Определение маршрутов (≥3 прохождения через ячейку)
- [ ] Клиент: отображение нити маршрута сквозь туман
